## 1. 项目概览

### 1.1 一句话定位

**"24 小时金融信息雷达"** — 自动聚合全球金融快讯、深度文章、加密货币动态，每 15 分钟更新一次，通过静态页面展示，零成本运行在 GitHub Pages 上。

### 1.2 目标覆盖范围

| 维度 | 覆盖内容 |
|------|----------|
| **市场** | A 股、港股、美股、全球宏观、外汇、大宗商品、加密货币 |
| **语言** | 中文源 + 英文源，双语标题支持 |
| **内容类型** | 实时快讯（7x24）、深度文章、社区讨论 |
| **数据源数量** | 内置采集 6 个 + OPML RSS 26 条（见 `feeds/finance.opml` 实际条目） |

### 1.3 核心架构继承

直接复用 ai-news-radar 的已验证架构：

```
Python 采集脚本 → JSON 数据文件 → 静态前端 → GitHub Actions 自动化
```

**为什么不重写？** 这套架构已在 ai-news-radar 中跑通：零 API Key 依赖、零服务器成本、Git 版本控制所有数据。金融新闻聚合本质上和 AI 新闻聚合是同类问题（采集→过滤→展示），架构完全通用。

## 4. 数据采集流水线

### 4.1 采集函数设计

每个内置源对应一个 `fetch_xxx()` 函数，统一返回 `list[RawItem]`：

```python
@dataclass
class RawItem:
    site_id: str          # 源标识，如 "wallstreetcn"
    site_name: str        # 展示名，如 "华尔街见闻"
    source: str           # 子分类，如 "全球快讯"
    title: str            # 标题
    url: str              # 原文链接
    published_at: datetime | None  # 发布时间
    meta: dict[str, Any]  # 扩展元数据（重要度、标签等）
```

### 4.2 各源采集逻辑

#### 华尔街见闻 (`fetch_wallstreetcn`)

```python
# 参数见基线表。Phase 1 channels: ["global-channel", "us-stock-channel", "a-stock-channel"]
# API: GET https://api-one.wallstcn.com/apiv1/content/lives
# Response: data.items[] → {id, display_time, title, content_text, channels[], symbols[]}

def fetch_wallstreetcn(session, now):
    channels = WALLSTREETCN_CHANNELS  # 从常量读取
    items = []
    for ch in channels:
        resp = session.get(BASE_URL, params={"channel": ch, "limit": WALLSTREETCN_LIMIT})
        for item in resp.json()["data"]["items"]:
            items.append(RawItem(
                site_id="wallstreetcn",
                site_name="华尔街见闻",
                source=ch.replace("-channel", ""),
                title=item["title"] or item.get("content_text", "")[:80],
                url=f"https://wallstreetcn.com/live/{item['id']}",
                published_at=parse_unix_timestamp(item["display_time"]),
                meta={"level": item.get("score", 0)},
            ))
    return items
```

## 5. 金融内容过滤体系

### 5.1 设计思路

ai-news-radar 用 `is_ai_related_record()` 过滤 AI 相关内容。金融版需要完全替换为 `is_finance_related_record()`。

但金融信息源与 AI 信息源有一个关键区别：**金融源本身已经是专业源**。华尔街见闻、金十数据出来的内容 99% 都是金融相关的，不需要二次过滤。过滤主要用于：
1. 从综合性源（如 CNBC、Reuters）中筛出金融相关内容
2. 排除明显噪音（广告、娱乐、体育）
3. 对内容进行市场分类（A 股 / 美股 / 港股 / 宏观 / 加密）

### 5.2 关键词归一化规则

> **约束**：所有关键词在定义时统一小写存储。匹配前将待检文本统一 `.lower()`。

```python
def classify_market(title: str, source: str) -> list[str]:
    """返回新闻所属市场标签列表，可能多个（如 "美联储降息" 同时属于 macro + us_stock）"""
    text = f"{title} {source}".lower()
    tags = []
    for tag, keywords in MARKET_TAGS.items():
        if any(kw.lower() in text for kw in keywords):
            tags.append(tag)
    return tags or ["general"]
```

### 5.5 过滤逻辑

```python
def is_finance_related_record(record):
    site_id = record.get("site_id", "")

    # 专业金融源直接放行
    if site_id in {"wallstreetcn", "jin10", "cls", "eastmoney", "gelonghui", "blockbeats"}:
        return True

    # 通用源需要关键词匹配
    title = record.get("title", "")
    source = record.get("source", "")
    text = f"{title} {source}".lower()

    # 先排除噪音
    if contains_any_keyword(text, NOISE_KEYWORDS):
        return False

    # 匹配金融关键词
    has_finance_zh = contains_any_keyword(text, FINANCE_KEYWORDS_ZH)
    has_finance_en = EN_FINANCE_SIGNAL_RE.search(text) is not None

    return has_finance_zh or has_finance_en
```

> **与 ai-news-radar 的关键区别**：ai-news-radar 的大部分源是综合性的（TechURLs, TopHub, NewsNow），所以需要较强的关键词过滤。金融版的内置源本身就是专业金融源，过滤主要作用于 RSS 订阅中的综合性英文源。

## 8. 部署与自动化

### 8.1 GitHub Actions Workflow

```yaml
name: Update Finance News Snapshot

on:
  workflow_dispatch:
  schedule:
    - cron: "*/15 * * * *"  # 每 15 分钟（金融信息时效性更强）

# P1-4 并发控制：同一 workflow 同时只运行一个实例，新实例取消正在运行的旧实例
concurrency:
  group: finance-news-update
  cancel-in-progress: true

permissions:
  contents: write

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"

      - name: Install dependencies
        run: pip install -r requirements.txt

      - name: Prepare OPML
        env:
          FINANCE_OPML_B64: ${{ secrets.FINANCE_OPML_B64 }}
        run: |
          mkdir -p feeds
          if [ -n "$FINANCE_OPML_B64" ]; then
            echo "$FINANCE_OPML_B64" | base64 --decode > feeds/finance.opml
          fi

      - name: Update data
        run: |
          if [ -f feeds/finance.opml ]; then
            python scripts/update_finance.py --output-dir data --window-hours 24 --rss-opml feeds/finance.opml
          else
            python scripts/update_finance.py --output-dir data --window-hours 24
          fi

      - name: Commit and push (with rebase retry)
        run: |
          if git diff --quiet; then exit 0; fi
          git config user.name "github-actions[bot]"
          git config user.email "41898282+github-actions[bot]@users.noreply.github.com"
          git add data/
          git commit -m "chore: update finance news snapshot"
          # push 冲突重试：最多 3 次，每次先 pull --rebase
          for i in 1 2 3; do
            git push && break
            echo "Push failed (attempt $i), rebasing..."
            git pull --rebase origin main
          done
```
